git add Cash.java - по одному добавлял классы чтобы гит мог их отслеживать
git add . - иногда во всех классах были мелкие фиксы и этой командой я добавлял их всех
git commit -m "Add new class Cash for payment" - каждую добавленую сущность комитил
git branch - смотрел в какой ветке я нахожусь чтобы потом правильно мержить
git checkout (название ветки) - переходил на нужную ветку
git status - смотрел что добавилось было так что пробел добавил и заметил только после git status
git merge master - обновлялся от мастера так как там видел более актуальный код чем в ветке home_work_25
git push -u origin master - я забыл что нужно было в home_work_25 пушить и запушил в мастер в удаленую ветку
